# Entire Replay Lab v0.1.42

Diff-inspection failure report stability.

## What Changed

- Replay now keeps saved reports schema-stable when post-run diff inspection
  fails.
- Diff-inspection failures are recorded as replay warnings while
  `changed_files` is preserved as an empty JSON array.
- Added focused Go coverage proving saved reports avoid `changed_files: null`
  on this failure path.
- Refreshed `patches/entire-replay-lab.patch` and the pinned patch SHA-256.
- Updated architecture, acceptance, testing, JSON schema, and reproducibility
  docs.

## Why It Matters

Diff inspection is diagnostic plumbing, so a failure there should not corrupt
the report contract. This release keeps replay evidence inspectable and
schema-valid even when changed-file collection cannot complete.

- This remains a prototype patch repo, not an upstream Entire CLI release.

  authenticated locally.
