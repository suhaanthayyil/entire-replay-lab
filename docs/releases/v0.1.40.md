# Entire Replay Lab v0.1.40

Semantic scoring failure warnings.

## What Changed

- Replay reports now save a warning when an installed `entire-sem` command fails
  or emits invalid JSON.
- Missing `entire-sem` remains a quiet optional-feature skip, but
  installed-but-broken semantic scoring is no longer indistinguishable from an
  absent tool.
- Added focused Go coverage proving semantic-scoring warnings persist in saved
  replay reports.
- Refreshed `patches/entire-replay-lab.patch` and the pinned patch SHA-256.
- Updated acceptance, architecture, testing, FAQ, and reproducibility docs.

## Why It Matters

Semantic similarity is optional, but an installed scoring tool that fails is
diagnostically different from a tool that is not installed. This release keeps
Replay Lab's graceful degradation while making semantic-scoring failures visible
in the saved evidence.

## Known Limitations

- This remains a prototype patch repo, not an upstream Entire CLI release.
- Live replay still requires each selected coding agent to be installed and
  authenticated locally.
- Pi is included in all-agent eval output, but skipped until a safe
  non-interactive launch contract is available.
