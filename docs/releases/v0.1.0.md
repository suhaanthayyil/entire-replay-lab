# Entire Replay Lab v0.1.0

First public prototype of Entire Replay Lab.

## What It Is

Replay Lab turns real Entire checkpoints into private coding-agent evals. It
checks out the checkpoint's base commit in an isolated worktree, reruns the
original prompt with a selected agent, and compares the result against the real
committed outcome.

## Why It Matters

Teams usually choose coding agents from generic benchmarks or vibes. Replay Lab
lets a team evaluate agents on its own repo, prompts, tests, and historical
work.

## Included

- self-contained patch against a known `entireio/cli` base commit
- `entire replay checkpoint <checkpoint-id>`
- `entire replay report <run-id>`
- `entire eval run`
- `entire eval report <run-id>`
- file-overlap, test, risk, duration, token, and optional semantic metrics
- JSON examples and local schemas
- dependency-free example/schema validation
- doctor, smoke, patch-check, release-check, build, and cleanup scripts
- MIT license

## Known Limitations

- This is still a prototype patch, not an upstream Entire CLI release.
- Live replay requires a launchable local agent and its own auth/model access.
- `--test-cmd` is explicit; v0.1.0 does not discover tests automatically.
- Semantic similarity is optional and requires `entire-sem`.
- Replay quality is judged against historical commits, not a proof of the only
  correct solution.
