# Entire Replay Lab v0.1.37

Missing-checkpoint eval matrix fidelity.

## What Changed

- Hardened `entire eval run` so checkpoint resolution failures still emit one
  failed replay row per selected agent.
- Missing-checkpoint rows preserve selected agent names, model overrides,
  stable empty arrays, skipped test status, and the checkpoint error.
- Eval summaries now stay agent-complete when a selected checkpoint cannot be
  resolved.
- Added focused Go coverage for the missing-checkpoint eval matrix behavior.
- Refreshed `patches/entire-replay-lab.patch` and the pinned patch SHA-256.
- Updated architecture, acceptance, testing, and reproducibility docs.

## Why It Matters

Eval reports should remain a matrix of requested agents by requested tasks,
even when a task cannot be built from a checkpoint. A single agentless failure
row hides which agents were part of the attempted eval and weakens summaries.
This release keeps failure reports shaped like eval reports, which makes them
easier to inspect and safer for automation.

## Known Limitations

- This remains a prototype patch repo, not an upstream Entire CLI release.
- Live replay still requires each selected coding agent to be installed and
  authenticated locally.
- Pi is included in all-agent eval output, but skipped until a safe
  non-interactive launch contract is available.
