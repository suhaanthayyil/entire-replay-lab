# Entire Replay Lab v0.1.30

Saved-report negative-path hardening.

## What Changed

- Extended `scripts/check-report-fixtures.sh` so missing replay report IDs must
  fail with a clear `read replay report` error.
- Extended the same fixture so missing eval report IDs must fail with a clear
  `read eval report` error.
- Checked both normal text mode and `--json` mode for replay and eval report
  lookups.
- Updated acceptance, testing, and command-reference docs to make this behavior
  part of the release contract.

## Why It Matters

Saved reports are the durable audit trail for Replay Lab runs. A missing or
mistyped report ID should fail loudly and explain which report file could not be
loaded, never produce empty or misleading output. This guard keeps that
diagnostic path stable for humans and automation.

  authenticated locally.
