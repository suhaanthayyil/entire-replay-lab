# Entire Replay Lab v0.1.63

Eval summary stale-value coverage.

## What Changed

- Broadened the eval-summary stale-value regression check in the local
  example/report validator.
- The validator now mutates every integer summary field present in the eval
  example, one field at a time, and requires summary consistency validation to
  reject each stale value.
- Covered run/status counts, pass rate, file averages, semantic averages,
  semantic run counts, duration, risk score, input tokens, and output tokens for
  each eval summary row.
- Updated acceptance, architecture, testing, command, and JSON schema docs.

## Why It Matters

Eval summaries are derived facts. A single stale count was already checked, but
that did not prove every derived field was protected. This release gives each
summary integer its own negative proof so regressions in rates, averages, risk,
duration, semantic metrics, or token totals fail during docs, smoke, CI, and
release validation.

## Known Limitations

- This remains a prototype patch repo, not an upstream Entire CLI release.
- Live replay still requires each selected coding agent to be installed and
  authenticated locally.
- Pi is included in all-agent eval output, but skipped until a safe
  non-interactive launch contract is available.
