# Entire Replay Lab v0.1.48

Legacy missing run status normalization.

## What Changed

- Replay report readers now normalize missing or empty top-level `status`
  values to `failed` before JSON output.
- Eval report readers apply the same conservative default to embedded replay
  runs before recomputing summaries.
- Direct JSON rendering also fills sparse replay runs with a failed status.
- Strengthened Go coverage for standalone replay reports, eval reports, and
  direct JSON rendering with missing run statuses.
- Refreshed `patches/entire-replay-lab.patch` and the pinned patch SHA-256.
- Updated architecture, acceptance, testing, JSON schema, and reproducibility
  docs.

## Why It Matters

The JSON schema requires each replay run to carry a valid status. Sparse legacy
reports without a run status could render as an empty string, while eval
summaries already treated those runs as failures. Normalizing the missing status
to `failed` keeps JSON schema-valid and keeps the rendered run rows aligned with
the recomputed summaries.
