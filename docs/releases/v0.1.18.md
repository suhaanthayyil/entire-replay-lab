# Entire Replay Lab v0.1.18

Schema validation for rendered Replay/Eval report JSON.

## What Changed

- Extended `scripts/validate-examples.py` with `--check <json> <schema>` for
  validating arbitrary Replay Lab JSON files against local schemas.
- Updated `scripts/check-report-fixtures.sh` to validate generated
  `entire replay report --json` and `entire eval report --json` output against
  the ReplayRun and ReplayEvalRun schemas.
- Aligned the `token_usage` schema/example with the actual Entire
  `agent.TokenUsage` JSON field names, including recursive subagent usage.
- Updated command, testing, and acceptance docs to describe schema-valid report
  fixture output.

## Why It Matters

Static examples can be correct while the built CLI drifts. This release closes
that gap by proving the actual report JSON emitted by the patched `entire`
binary still matches the same schema contract used by docs and examples.
