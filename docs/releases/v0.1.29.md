# Entire Replay Lab v0.1.29

CLI negative-path hardening for replay agent selection.

## What Changed

- Hardened `scripts/check-command-surface.sh` so the built binary must reject
  `entire replay checkpoint --agent all` with the intended eval-only message.
- Added a command-surface check that unknown replay agents fail before any repo
  lookup, with a clear `unknown replay agent` error.
- Extended `scripts/check-all-agent-eval.sh` to run an explicit
  `--agent unknown-agent` eval against a real checkpoint fixture and validate
  the resulting schema-valid skipped JSON run.

## Why It Matters

Replay Lab is meant to be run on real repositories and real agent setups, where
bad flags are common. These checks prove the CLI explains invalid agent choices
cleanly and keeps eval JSON stable even when a requested agent is unknown.

## Known Limitations

- This remains a prototype patch repo, not an upstream Entire CLI release.
- Live replay still requires each selected coding agent to be installed and
  authenticated locally.
- Pi is included in all-agent eval output, but skipped until a safe
  non-interactive launch contract is available.
