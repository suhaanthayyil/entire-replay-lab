# Entire Replay Lab v0.1.46

Skipped test command evidence.

## What Changed

- Skipped test rows now preserve the requested `test.command` when replay setup,
  agent execution, checkpoint resolution, or agent availability prevents the
  command from running.
- Added a shared skipped-test initializer for synthetic replay/eval rows.
- Strengthened Go coverage for setup failure, agent failure, missing checkpoint,
  and missing-agent-command paths.
- Refreshed `patches/entire-replay-lab.patch` and the pinned patch SHA-256.
- Updated architecture, acceptance, testing, JSON schema, and reproducibility
  docs.

## Why It Matters

When a user supplies `--test-cmd`, the command is part of the replay attempt
even if an earlier failure prevents execution. Keeping it in skipped test rows
makes saved reports more useful for auditing what was requested.

## Known Limitations

- This remains a prototype patch repo, not an upstream Entire CLI release.
- Live replay still requires each selected coding agent to be installed and
  authenticated locally.
- Pi is included in all-agent eval output, but skipped until a safe
  non-interactive launch contract is available.
