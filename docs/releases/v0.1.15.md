# Entire Replay Lab v0.1.15

Portability validation for reusable docs and scripts.

## What Changed

- Added `scripts/validate-portability.py`.
- Removed personal local paths from demo and testing docs.
- Made `scripts/demo-commands.sh` resolve `bin/entire` from the repo root.
- Made `scripts/refresh-patch.sh` require an explicit `ENTIRE_CLI_SOURCE`
  instead of assuming a local checkout path.
- Wired portability validation into `verify-repo`, release checks, CI, Makefile,
  and command/testing/release docs.

## Why It Matters

Replay Lab is public and should work cleanly on someone else's laptop. The
previous docs and helper scripts still had a few machine-specific paths from the
development machine. This release turns portability into a checked invariant so
the repo remains usable for Thomas, Entire, or any reviewer without path edits.
