# Entire Replay Lab v0.1.36

Test-command timeout evidence.

## What Changed

- Added optional `test.error` to ReplayRun JSON reports.
- Replay test-command timeouts now record a structured timeout cause while
  preserving partial test output.
- Text reports now render the test error separately from test output.
- Updated the ReplayRun schema, eval example payload, JSON-shape docs,
  architecture, acceptance, and testing docs.
- Added focused Go coverage for timed-out test command evidence.
- Refreshed `patches/entire-replay-lab.patch` and the pinned patch SHA-256.

## Why It Matters

Replay Lab is diagnostic. A test command that times out should not look like a
generic failed process with only an exit code. This release makes timeout cause
and partial test output available together, so saved reports explain what
happened without hiding the useful output produced before the deadline.

## Known Limitations

- This remains a prototype patch repo, not an upstream Entire CLI release.
- Live replay still requires each selected coding agent to be installed and
  authenticated locally.
- Pi is included in all-agent eval output, but skipped until a safe
  non-interactive launch contract is available.
