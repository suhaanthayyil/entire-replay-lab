# Entire Replay Lab v0.1.50

Legacy missing report ID recovery.

## What Changed

- Replay report readers now recover missing top-level report IDs from the
  sanitized report filename.
- Eval report readers apply the same recovery for missing top-level eval IDs.
- Recovered IDs are preserved in subsequent JSON output.
- Strengthened Go coverage for standalone replay reports and eval reports with
  missing top-level IDs.
- Refreshed `patches/entire-replay-lab.patch` and the pinned patch SHA-256.
- Updated architecture, acceptance, testing, JSON schema, and reproducibility
  docs.

## Why It Matters

The report filename is already the stable lookup identity for saved replay and
eval reports. When older sparse reports omit the top-level `id`, rendering that
empty field makes the report harder to audit and cross-reference. Recovering the
ID from the filename keeps legacy report output self-identifying without
inventing a new random value.
