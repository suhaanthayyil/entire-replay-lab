# Entire Replay Lab v0.1.49

Legacy unknown status normalization.

## What Changed

- Replay report readers now normalize unknown non-empty replay `status` values
  to `failed` before JSON output.
- Unknown non-empty `test.status` values now normalize to `skipped`.
- The original unknown status strings are preserved as replay warnings for
  auditability.
- Eval summaries are recomputed after normalization so summary counts match the
  rendered run rows.
- Strengthened Go coverage for standalone replay reports and eval reports with
  unknown legacy statuses.
- Refreshed `patches/entire-replay-lab.patch` and the pinned patch SHA-256.
- Updated architecture, acceptance, testing, JSON schema, and reproducibility
  docs.

## Why It Matters

The JSON schema only accepts known replay and test statuses. Sparse legacy
reports are now covered by v0.1.47 and v0.1.48; this release closes the
remaining non-empty legacy status case without losing the original value. A
downstream reader gets schema-valid status fields plus warnings that explain
what was normalized.

## Known Limitations

- This remains a prototype patch repo, not an upstream Entire CLI release.
- Live replay still requires each selected coding agent to be installed and
  authenticated locally.
- Pi is included in all-agent eval output, but skipped until a safe
  non-interactive launch contract is available.
