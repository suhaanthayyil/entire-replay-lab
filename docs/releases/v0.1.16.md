# Entire Replay Lab v0.1.16

Helper script hygiene and command-reference coverage validation.

## What Changed

- Added `scripts/validate-script-hygiene.py`.
- The validator checks helper script shebangs, executable bits, shell safety
  flags, Python CLI entrypoints, and command-reference coverage.
- Added missing command-reference sections for `demo-commands.sh`,
  `release-check.sh`, `replay-lab-env.sh`, `validate-doc-links.py`, and the new
  script hygiene validator.
- Wired script hygiene validation into `verify-repo`, release checks, CI,
  Makefile, and command/testing/release docs.

## Why It Matters

Replay Lab is script-heavy by design. A missing executable bit, stale command
reference, or unsafe shell helper can make the prototype feel broken even when
the Replay Lab patch itself is fine. This release makes script quality a checked
part of the release gate.
