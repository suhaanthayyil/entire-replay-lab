# Entire Replay Lab v0.1.10

Latest release tag commit verification for Replay Lab releases.

## What Changed

- `scripts/verify-release-state.sh` now checks that the latest release tag
  points at the current local `HEAD`.
- It also checks that the matching `origin` tag resolves to the same commit.
- `scripts/validate-release-docs.py` now sorts release files by semver, so
  `v0.1.10` and later validate correctly.
- `scripts/check-command-surface.sh` now captures help output before searching,
  avoiding false failures from `grep -q` and `pipefail`.
- Release docs now describe the stricter post-release verification contract.

## Why It Matters

The previous release-state verifier proved that release-note files, local tags,
and GitHub releases had matching names. This release also proves the latest tag
actually points at the commit being released, which catches stale tags or
incorrect post-release verification.
