# Entire Replay Lab v0.1.59

Closed object schema hardening.

## What Changed

- Added schema-driven negative checks for `additionalProperties: false` in the
  local example/report validator.
- The checks walk local and cross-schema `$ref` links, then mutate every present
  closed object in the replay/eval examples with an undocumented field.
- Covered nested specs, tests, metrics, token usage, eval summaries, and
  embedded replay runs inside eval reports, not just top-level payloads.
- Updated acceptance, architecture, testing, command, and JSON schema docs.

## Why It Matters

Closed schemas are what keep report JSON stable for scripts, fixtures, release
checks, and downstream tooling. A root payload can be closed while nested
objects quietly start accepting stray fields. This release makes closure a
checked invariant throughout the example report tree.

- Live replay still requires each selected coding agent to be installed and
  authenticated locally.

  non-interactive launch contract is available.
