# Entire Replay Lab v0.1.45

Empty eval agent preservation.

## What Changed

- Eval report normalization now preserves selected `agents` when an eval has an
  empty `runs` array.
- Sparse legacy eval reports still deduplicate agents and emit stable empty
  `summaries` and `runs` arrays.
- Agent lists are still derived from embedded replay runs when run rows exist
  and the saved agent set is missing or mismatched.
- Added focused Go coverage for legacy empty-run eval reports with selected
  agents.
- Refreshed `patches/entire-replay-lab.patch` and the pinned patch SHA-256.
- Updated architecture, acceptance, testing, JSON schema, and reproducibility
  docs.

## Why It Matters

An empty eval report can still carry useful selection metadata. The previous
normalization repair path could erase that agent list because there were no run
rows to derive from. This release keeps sparse reports informative while
preserving the schema-stable empty arrays.

## Known Limitations

- This remains a prototype patch repo, not an upstream Entire CLI release.
- Live replay still requires each selected coding agent to be installed and
  authenticated locally.
- Pi is included in all-agent eval output, but skipped until a safe
  non-interactive launch contract is available.
