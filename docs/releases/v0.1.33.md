# Entire Replay Lab v0.1.33

Replay timeout validation hardening.

## What Changed

- Hardened `entire replay checkpoint --timeout` and `entire eval run --timeout`
  so explicit `0` or negative durations fail before agent lookup or checkpoint
  discovery.
- Added focused Go coverage for replay and eval timeout validation.
- Added binary-level command-surface coverage proving the built CLI rejects
  non-positive replay and eval timeout values.
- Refreshed `patches/entire-replay-lab.patch` and the pinned patch SHA-256.
- Updated acceptance, testing, command-reference, and reproducibility docs.

## Why It Matters

Replay Lab launches external coding-agent processes. A mistyped timeout such as
`--timeout 0s` should not silently remove the timeout guard and create an
unbounded agent run. This release makes the safe behavior explicit and proves it
at both the Go unit-test layer and the built-binary command layer.

## Known Limitations

- This remains a prototype patch repo, not an upstream Entire CLI release.
- Live replay still requires each selected coding agent to be installed and
  authenticated locally.
- Pi is included in all-agent eval output, but skipped until a safe
  non-interactive launch contract is available.
