# Entire Replay Lab v0.1.35

Semantic scratch-index isolation.

## What Changed

- Hardened optional semantic scoring to build its comparison commit from a
  copied temporary git index.
- Replaced the previous temporary real commit plus `reset --mixed` cleanup with
  an unreachable `commit-tree` result, leaving replay worktree `HEAD` untouched.
- Semantic scoring now includes staged and untracked replay output without
  mutating the kept worktree's real index state.
- Added focused Go coverage for the semantic isolation behavior.
- Refreshed `patches/entire-replay-lab.patch` and the pinned patch SHA-256.
- Updated architecture, acceptance, testing, and reproducibility docs.

## Why It Matters

Kept replay worktrees are evidence. Semantic scoring should improve a report,
not rewrite the evidence while computing it. This release keeps optional
`entire-sem` comparison faithful to the replay output while preserving the exact
inspection state an agent left behind.

## Known Limitations

- This remains a prototype patch repo, not an upstream Entire CLI release.
- Live replay still requires each selected coding agent to be installed and
  authenticated locally.
- Pi is included in all-agent eval output, but skipped until a safe
  non-interactive launch contract is available.
