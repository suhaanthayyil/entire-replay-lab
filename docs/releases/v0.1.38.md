# Entire Replay Lab v0.1.38

Cleanup-failure worktree path fidelity.

## What Changed

- Replay reports now preserve `worktree_path` when automatic cleanup fails,
  even if `--keep-worktree` was not requested.
- Cleanup warnings are still saved with the report, so the leaked worktree path
  and removal failure are visible together.
- Added focused Go coverage for cleanup-failure report persistence.
- Refreshed `patches/entire-replay-lab.patch` and the pinned patch SHA-256.
- Updated architecture, acceptance, testing, and reproducibility docs.

## Why It Matters

A failed automatic cleanup is rare, but when it happens the surviving worktree
is the artifact operators need most. Without the path in saved JSON, automation
could see the warning but still lack the concrete directory to inspect or
remove. This release keeps cleanup failures self-describing in the report.

## Known Limitations

- This remains a prototype patch repo, not an upstream Entire CLI release.
- Live replay still requires each selected coding agent to be installed and
  authenticated locally.
- Pi is included in all-agent eval output, but skipped until a safe
  non-interactive launch contract is available.
