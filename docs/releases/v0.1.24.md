# Entire Replay Lab v0.1.24

Runtime registry-backed all-agent eval expansion.

## What Changed

- Made `entire eval run --agent all` derive from Entire's user-facing built-in
  coder registry at runtime instead of a separate hardcoded Replay Lab list.
- Kept launchable replay agents first in eval output, then sorted the remaining
  registered coders for deterministic reports.
- Added tests for registry-derived expansion, launchable-first ordering, and
  exclusion of internal test-only agents.
- Added a build lock so concurrent smoke, release, and manual build runs cannot
  mutate the shared patched CLI checkout at the same time.
- Updated the all-agent fixture, command docs, doctor output, patch artifact,
  and pinned patch hash.

## Why It Matters

Replay Lab now tracks the current Entire CLI build more directly. If Entire adds
or removes a user-facing built-in coder, `--agent all` follows the registry
instead of waiting for a second hardcoded list to be edited.

The build lock also protects the reproducibility workflow: heavy verification
scripts can now be launched at the same time without resetting the shared
patched checkout under each other.
