# Entire Replay Lab v0.1.61

Required field schema hardening.

## What Changed

- Added schema-driven negative checks for JSON Schema `required` constraints in
  the local example/report validator.
- The checks walk local and cross-schema `$ref` links, remove each present
  required key one at a time, and require validation to fail.
- Covered required keys at the replay/eval roots plus nested specs, tests,
  metrics, eval summaries, and embedded replay runs inside eval reports.
- Updated acceptance, architecture, testing, command, and JSON schema docs.

## Why It Matters

Required fields are the spine of the report contract. A schema can still reject
bad values while accidentally allowing sparse reports that omit mandatory
sections such as specs, tests, metrics, or summaries. This release makes missing
required fields a checked invariant throughout the example report tree.
