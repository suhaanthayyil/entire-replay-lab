# Entire Replay Lab v0.1.31

Eval limit and patch-refresh hardening.

## What Changed

- Hardened `entire eval run --from-checkpoints --limit` so `0` or negative
  values fail before checkpoint discovery.
- Added command-surface coverage proving the built binary rejects a
  non-positive `--from-checkpoints --limit`.
- Fixed `scripts/refresh-patch.sh` so patch regeneration diffs the current
  local CLI checkout against the pinned upstream base and includes new Replay
  Lab files without mutating the checkout's real git index.
- Refreshed `patches/entire-replay-lab.patch` and the pinned patch SHA-256.
- Updated acceptance, testing, command-reference, and reproducibility docs.

## Why It Matters

`--from-checkpoints --limit 0` is easy to type while experimenting. Before this
release, that could widen an eval to every available checkpoint. This release
makes the safe behavior explicit: non-positive limits are rejected before any
checkpoint scan. The refresh-script fix also protects the release artifact by
making local patch regeneration work from the normal dirty patched checkout
state.

## Known Limitations

- This remains a prototype patch repo, not an upstream Entire CLI release.
- Live replay still requires each selected coding agent to be installed and
  authenticated locally.
- Pi is included in all-agent eval output, but skipped until a safe
  non-interactive launch contract is available.
