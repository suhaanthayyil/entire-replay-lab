# Entire Replay Lab v0.1.54

Unique report evidence arrays.

## What Changed

- The dependency-free schema validator now supports JSON Schema `uniqueItems`
  checks.
- Replay file-list arrays and eval `agents` now reject duplicate entries in the
  schema contract.
- Added validator negative checks that mutate unique arrays to contain
  duplicates and prove those payloads are rejected.
- Updated acceptance, architecture, testing, command, and JSON schema docs.

## Why It Matters

Replay reports use file lists and selected-agent lists as evidence, not as
ordered logs. Duplicate entries make comparisons noisier without adding facts,
and eval agent duplicates can make summaries look less trustworthy. This
release turns that set-like behavior into a checked schema invariant for docs,
smoke, CI, and release validation.

## Known Limitations

- This remains a prototype patch repo, not an upstream Entire CLI release.
- Live replay still requires each selected coding agent to be installed and
  authenticated locally.
- Pi is included in all-agent eval output, but skipped until a safe
  non-interactive launch contract is available.
