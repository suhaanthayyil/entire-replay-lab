# Entire Replay Lab v0.1.47

Legacy missing test status normalization.

## What Changed

- Replay report readers now normalize missing or empty `test.status` values to
  `skipped` before JSON output.
- Eval report readers apply the same normalization to embedded replay runs.
- Direct JSON rendering also fills sparse test objects with a skipped status.
- Strengthened Go coverage for standalone replay reports, eval reports, and
  direct JSON rendering with legacy sparse test objects.
- Refreshed `patches/entire-replay-lab.patch` and the pinned patch SHA-256.
- Updated architecture, acceptance, testing, JSON schema, and reproducibility
  docs.

## Why It Matters

The JSON schema requires every replay test object to carry a valid status.
Older sparse reports, or direct rendering of zero-value test objects, could emit
an empty status string. Normalizing those cases to `skipped` keeps reports
schema-valid and makes it clear no test command actually ran.
