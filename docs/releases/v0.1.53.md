# Entire Replay Lab v0.1.53

Non-blank identity schema validation.

## What Changed

- The dependency-free schema validator now supports JSON Schema `pattern`
  checks for strings.
- Replay and eval schemas now reject whitespace-only required identity strings,
  including report IDs, agents, checkpoint IDs, prompts, and commit anchors.
- Added validator negative checks that mutate required identity strings to
  blank whitespace values and prove those payloads are rejected.
- Updated acceptance, architecture, testing, command, and JSON schema docs.

## Why It Matters

v0.1.52 made required identity strings non-empty. Whitespace-only strings still
pass that rule while carrying no useful identity. This release closes that
remaining gap so local schema validation proves those fields are meaningful
enough to name, route, compare, and audit replay/eval reports.

## Known Limitations

- This remains a prototype patch repo, not an upstream Entire CLI release.
- Live replay still requires each selected coding agent to be installed and
  authenticated locally.
- Pi is included in all-agent eval output, but skipped until a safe
  non-interactive launch contract is available.
