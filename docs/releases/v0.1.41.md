# Entire Replay Lab v0.1.41

Worktree setup failure reports.

## What Changed

- Replay now saves a failed report when isolated worktree setup fails after a
  checkpoint spec has been resolved.
- Setup-failure reports preserve the run ID, spec, agent, model override,
  skipped test status, stable empty `changed_files`, and setup error.
- The replay agent is not launched when setup fails.
- Added focused Go coverage for persisted worktree setup failures.
- Refreshed `patches/entire-replay-lab.patch` and the pinned patch SHA-256.
- Updated architecture, acceptance, testing, and reproducibility docs.

## Why It Matters

Replay Lab is most useful when every attempted replay leaves evidence. A
worktree setup failure used to return before writing a report, even though the
checkpoint, target agent, and model were already known. This release keeps those
failures visible and inspectable under `.git/entire-replay/runs/`.

## Known Limitations

- This remains a prototype patch repo, not an upstream Entire CLI release.
- Live replay still requires each selected coding agent to be installed and
  authenticated locally.
- Pi is included in all-agent eval output, but skipped until a safe
  non-interactive launch contract is available.
