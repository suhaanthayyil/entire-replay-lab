# Entire Replay Lab v0.1.27

Expanded launchable Entire coder support.

## What Changed

- Added native replay runners for Copilot CLI, Cursor Agent CLI, Factory AI
  Droid, and OpenCode.
- Kept the existing Claude Code, Codex, and Gemini CLI runners.
- `entire eval run --agent all` now puts seven launchable Entire coders first:
  `claude-code`, `codex`, `copilot-cli`, `cursor`, `factoryai-droid`,
  `gemini`, and `opencode`.
- Pi remains represented in all-agent evals as a skipped row until Pi exposes a
  safe non-interactive launch contract.
- Updated command docs, architecture docs, FAQ, testing docs, doctor checks,
  examples, the all-agent fixture, patch artifact, and pinned patch hash.

## Why It Matters

Replay Lab is more useful when it can run the same checkpoint task across the
actual coding agents a team is trying. This release moves the MVP from
three-agent replay to broad local runner coverage across the current documented
Entire coder set, while still being honest about Pi instead of binding to an
unsafe `pi` binary name.

## Known Limitations

- This remains a prototype patch repo, not an upstream Entire CLI release.
- Live replay still requires each selected coding agent to be installed and
  authenticated locally.
- Pi is included in all-agent eval output, but skipped until a safe
  non-interactive launch contract is available.
