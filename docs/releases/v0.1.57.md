# Entire Replay Lab v0.1.57

Timestamp ordering consistency.

## What Changed

- The local example/report validator now rejects replay/eval timestamps where
  `finished_at` is earlier than `started_at`.
- The check covers standalone replay reports, eval wrapper timestamps, and
  embedded replay runs inside eval reports.
- Added validator negative checks that invert finish times and prove those
  payloads are rejected.
- Updated acceptance, architecture, testing, command, and JSON schema docs.

## Why It Matters

v0.1.56 made timestamps syntactically valid RFC3339 strings. A timestamp can be
well-formed while still claiming a report finished before it started. This
release catches that temporal inconsistency during the same local validation
used by docs, smoke, CI, and release checks.

## Known Limitations

- This remains a prototype patch repo, not an upstream Entire CLI release.
- Live replay still requires each selected coding agent to be installed and
  authenticated locally.
- Pi is included in all-agent eval output, but skipped until a safe
  non-interactive launch contract is available.
