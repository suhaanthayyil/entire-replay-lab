# Entire Replay Lab v0.1.44

Eval summary repair.

## What Changed

- Eval report summaries are now recomputed from embedded replay runs during
  save, read, and JSON output normalization.
- Legacy eval reports with missing or stale summaries are repaired before
  `entire eval report --json` renders them.
- Missing or mismatched eval `agents` arrays are derived from embedded replay
  runs while valid existing agent lists are preserved.
- Added focused Go coverage for legacy eval reports with null summaries and
  stale summary totals.
- Refreshed `patches/entire-replay-lab.patch` and the pinned patch SHA-256.
- Updated architecture, acceptance, testing, JSON schema, and reproducibility
  docs.

## Why It Matters

Eval summaries are derived data. If they drift from the embedded runs, report
consumers can rank agents incorrectly even though the detailed run rows are
accurate. This release makes report rendering repair that drift automatically.

## Known Limitations

- This remains a prototype patch repo, not an upstream Entire CLI release.
- Live replay still requires each selected coding agent to be installed and
  authenticated locally.
- Pi is included in all-agent eval output, but skipped until a safe
  non-interactive launch contract is available.
