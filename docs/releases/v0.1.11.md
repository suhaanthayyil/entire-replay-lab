# Entire Replay Lab v0.1.11

Latest GitHub release body verification for Replay Lab releases.

## What Changed

- `scripts/verify-release-state.sh` now compares the latest GitHub release body
  with the matching `docs/releases/<version>.md` file.
- Mismatches print a unified diff so release-note drift is easy to repair.
- Release docs now mention the body comparison as part of post-release
  verification.

## Why It Matters

The release-state verifier already proved that release names, tags, and latest
tag commits match. This release also proves the human-facing GitHub release
notes match the checked-in release notes, preventing quiet documentation drift
after publishing.
