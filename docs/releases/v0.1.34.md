# Entire Replay Lab v0.1.34

Kept-worktree diff hygiene.

## What Changed

- Hardened replay diff collection to use a copied temporary git index.
- Replay Lab still captures untracked agent output in `changed_files` and the
  saved diff.
- Kept replay worktrees no longer inherit intent-to-add index entries from diff
  collection, so untracked files remain naturally visible as untracked.
- Added focused Go coverage for the kept-worktree index hygiene behavior.
- Refreshed `patches/entire-replay-lab.patch` and the pinned patch SHA-256.
- Updated architecture, acceptance, testing, and reproducibility docs.

## Why It Matters

`--keep-worktree` exists so a user can inspect what an agent actually did. The
old diff path used `git add -N .` in that worktree so untracked files appeared
in the diff, but that left intent-to-add index state behind. This release keeps
the same diff fidelity while leaving the kept worktree easier to inspect.

## Known Limitations

- This remains a prototype patch repo, not an upstream Entire CLI release.
- Live replay still requires each selected coding agent to be installed and
  authenticated locally.
- Pi is included in all-agent eval output, but skipped until a safe
  non-interactive launch contract is available.
