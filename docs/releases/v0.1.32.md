# Entire Replay Lab v0.1.32

Patch-refresh regression gate.

## What Changed

- Added `scripts/check-refresh-patch.sh`, a regression check for
  `scripts/refresh-patch.sh`.
- The new check applies the checked-in patch to a fresh temporary CLI clone,
  refreshes a patch from that dirty checkout, and verifies the refreshed patch
  byte-matches the checked-in artifact.
- The same check verifies the source checkout's git status is unchanged, so
  refreshes cannot silently stage Replay Lab's untracked upstream files.
- Wired the new proof into smoke, release checks, GitHub Actions CI, repo
  verification, acceptance docs, testing docs, and command-reference docs.

## Why It Matters

The patch file is the artifact that makes Replay Lab reproducible. If the
refresh workflow misses untracked files or mutates the source checkout index,
the next release can look locally correct while publishing a broken or
surprising patch. This release turns that workflow into a repeatable gate.

## Known Limitations

- This remains a prototype patch repo, not an upstream Entire CLI release.
- Live replay still requires each selected coding agent to be installed and
  authenticated locally.
- Pi is included in all-agent eval output, but skipped until a safe
  non-interactive launch contract is available.
