# Entire Replay Lab v0.1.58

Numeric range schema hardening.

## What Changed

- Added schema-driven negative checks for JSON Schema `minimum` and `maximum`
  constraints in the local example/report validator.
- The checks cover replay reports, eval summaries, and embedded eval replay
  runs wherever bounded integers appear in the examples.
- Numeric regressions now fail locally if non-negative durations, token counts,
  run counts, risk scores, file overlaps, or 0-100 score/rate bounds are
  removed or ignored.
- Updated acceptance, architecture, testing, command, and JSON schema docs.

## Why It Matters

Replay and eval reports use numeric fields for timing, scoring, risk, and token
evidence. A schema that accidentally accepts negative counts or scores above
100 can make comparisons misleading while still looking structurally valid.
This release turns those numeric bounds into checked invariants for docs,
smoke, CI, and release validation.

## Known Limitations

- This remains a prototype patch repo, not an upstream Entire CLI release.
- Live replay still requires each selected coding agent to be installed and
  authenticated locally.
- Pi is included in all-agent eval output, but skipped until a safe
  non-interactive launch contract is available.
