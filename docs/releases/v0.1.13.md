# Entire Replay Lab v0.1.13

Markdown fenced code block validation for Replay Lab docs.

## What Changed

- Added `scripts/validate-markdown-fences.py`.
- The validator checks that fenced code blocks are closed and use expected
  languages.
- JSON fenced blocks are parsed as JSON.
- Wired the validator into `verify-repo`, release checks, CI, Makefile, and
  command/testing/release docs.

## Why It Matters

Replay Lab depends heavily on README and docs examples. Link checks catch broken
targets, but they do not catch malformed fences or invalid JSON snippets. This
release adds a lightweight docs syntax gate so examples stay readable and
machine-checkable.
