# Entire Replay Lab v0.1.56

Timestamp schema validation.

## What Changed

- The dependency-free schema validator now supports JSON Schema
  `format: date-time` checks for RFC3339 timestamps.
- Replay and eval `started_at`/`finished_at` fields now reject malformed
  timestamp strings in the schema contract.
- Added validator negative checks for standalone replay timestamps, eval
  wrapper timestamps, and embedded eval replay timestamps.
- Updated acceptance, architecture, testing, command, and JSON schema docs.

## Why It Matters

Replay reports use timestamps to explain ordering, duration, and audit context.
Plain string validation lets malformed timestamps pass as if they were usable
time evidence. This release makes replay and eval timestamps machine-checkable
through the same local schema path used by docs, smoke, CI, and release checks.

## Known Limitations

- This remains a prototype patch repo, not an upstream Entire CLI release.
- Live replay still requires each selected coding agent to be installed and
  authenticated locally.
- Pi is included in all-agent eval output, but skipped until a safe
  non-interactive launch contract is available.
