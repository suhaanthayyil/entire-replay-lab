# Entire Replay Lab v0.1.52

Required string schema hardening.

## What Changed

- The dependency-free schema validator now supports `minLength`.
- Replay and eval schemas now reject empty required identity strings, including
  report IDs, agents, checkpoint IDs, prompts, and commit anchors.
- Added validator negative checks that mutate required identity strings to
  empty values and prove those payloads are rejected.
- Updated acceptance, architecture, testing, command, and JSON schema docs.

## Why It Matters

An empty string can satisfy a plain JSON Schema `type: string` rule while still
being useless as an ID, agent name, checkpoint reference, prompt, or commit
anchor. This release makes required report identity fields prove they are
present and non-empty during the same local validation used by docs, smoke, CI,
and release checks.

## Known Limitations

- This remains a prototype patch repo, not an upstream Entire CLI release.
- Live replay still requires each selected coding agent to be installed and
  authenticated locally.
- Pi is included in all-agent eval output, but skipped until a safe
  non-interactive launch contract is available.
