# Entire Replay Lab v0.1.21

Binary-level all-agent eval verification.

## What Changed

- Added `scripts/check-all-agent-eval.sh`.
- The new fixture creates a temporary git repo, commits a real
  `Entire-Checkpoint` trailer, writes checkpoint metadata through Entire's own
  checkpoint writer, then runs `entire eval run --agent all --json` with live
  agent binaries hidden from `PATH`.
- The fixture validates that all nine built-in Entire coders are present in the
  generated eval JSON and rendered report.
- Fixed synthetic skipped/failed eval runs so JSON emits stable empty arrays
  instead of `null` for `changed_files` and `files_touched`.
- Wired the new fixture into `release-check.sh`, `smoke.sh`, command docs, and
  repo verification.

## Why It Matters

The previous release proved `--agent all` through Go tests and manual checks.
This release turns that into a binary-level gate. It caught a real schema issue:
skipped eval rows from the actual command could emit `changed_files: null`,
which violated the documented JSON contract. Now the release path proves the
actual command emits schema-valid all-agent eval output without launching live
models.
