# Entire Replay Lab v0.1.39

Independent replay and test timeout budgets.

## What Changed

- Replay setup, agent execution, and optional test commands now receive
  independent timeout contexts derived from `--timeout`.
- A slow but successful replay agent no longer shortens the optional test
  command's timeout window.
- Added focused Go coverage for separate agent and test-command timeout budgets.
- Refreshed `patches/entire-replay-lab.patch` and the pinned patch SHA-256.
- Updated command, architecture, acceptance, testing, and reproducibility docs.

## Why It Matters

`--timeout` is a safety rail for each expensive step, not a shared stopwatch for
the whole replay. The previous shared context meant an agent that finished just
before its deadline could leave the test command only a sliver of time, causing
misleading test failures. This release makes the documented timeout contract
match runtime behavior.

## Known Limitations

- This remains a prototype patch repo, not an upstream Entire CLI release.
- Live replay still requires each selected coding agent to be installed and
  authenticated locally.
- Pi is included in all-agent eval output, but skipped until a safe
  non-interactive launch contract is available.
