# Entire Replay Lab v0.1.60

Literal schema hardening.

## What Changed

- Added schema-driven negative checks for JSON Schema `const` and `enum`
  constraints in the local example/report validator.
- The checks walk local and cross-schema `$ref` links, then mutate each present
  literal-constrained field to an invalid value and require validation to fail.
- Covered replay/eval `schema_version` constants plus replay and test status
  enums in standalone replay reports and embedded eval replay runs.
- Updated acceptance, architecture, testing, command, and JSON schema docs.

## Why It Matters

Report consumers need `schema_version` and status values to be predictable.
Structural validation is not enough if a report can silently claim an unknown
schema version or status. This release turns those literal contracts into
checked invariants for docs, smoke, CI, and release validation.

## Known Limitations

- This remains a prototype patch repo, not an upstream Entire CLI release.
- Live replay still requires each selected coding agent to be installed and
  authenticated locally.
- Pi is included in all-agent eval output, but skipped until a safe
  non-interactive launch contract is available.
