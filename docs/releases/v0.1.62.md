# Entire Replay Lab v0.1.62

Type schema hardening.

## What Changed

- Added schema-driven negative checks for JSON Schema `type` constraints in the
  local example/report validator.
- The checks walk local and cross-schema `$ref` links, replace each present
  typed value with a wrong JSON value kind, and require validation to fail.
- Covered root payloads, nested objects, arrays, strings, integers, booleans,
  eval summaries, token usage, and embedded replay runs inside eval reports.
- Updated acceptance, architecture, testing, command, and JSON schema docs.

## Why It Matters

Report data needs stable JSON value kinds, not just valid field names. A metric
that turns into a string, a boolean that turns into text, or an array that turns
into an object can break consumers even if the field is still present. This
release makes type correctness a checked invariant throughout the example
report tree.
