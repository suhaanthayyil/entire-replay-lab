# Entire Replay Lab v0.1.2

Reproducibility hardening for the Replay Lab prototype.

## What Changed

- Added `scripts/replay-lab-env.sh` as the shared source of truth for the pinned
  Entire CLI repo, base commit, and patch path.
- Updated `build-cli.sh`, `check-patch.sh`, and `refresh-patch.sh` to source the
  shared build inputs.
- Made `refresh-patch.sh` default to diffing against the pinned
  `ENTIRE_CLI_REF` instead of moving `origin/main`.
- Added `scripts/verify-reproducibility.sh`.
- Wired reproducibility verification into repo verification, release checks, CI,
  and the Makefile.
- Added `docs/REPRODUCIBILITY.md`.

## Why It Matters

Replay Lab is currently shipped as a patch against a known Entire CLI base. This
release makes that base explicit and machine-checked, reducing the chance that
docs, build scripts, and patch tests drift apart.
