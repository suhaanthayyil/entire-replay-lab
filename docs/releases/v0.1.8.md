# Entire Replay Lab v0.1.8

Binary-level report fixture verification for Replay Lab.

## What Changed

- Added `scripts/check-report-fixtures.sh`.
- Added `make report-fixtures`.
- Wired report fixture verification into release checks, smoke checks, and CI.
- The verifier creates a temporary git repo, seeds example replay/eval JSON into
  the git common dir, and runs the built `entire` binary's report commands.
- Both `entire replay report` and `entire eval report` are checked in text and
  `--json` modes.
- Updated README, command, acceptance, testing, and release docs.

## Why It Matters

Help output proves the commands exist, but users rely on saved reports after a
replay or eval run. This release proves the built binary can read stored result
JSON from the expected git-common-dir layout and render it back in both human
and machine-readable formats.
