# Entire Replay Lab v0.1.20

All-coder eval MVP and eval summary consistency validation.

## What Changed

- Added eval summary consistency checks to `scripts/validate-examples.py`.
- Added `entire eval run --agent all` support across every built-in Entire
  coder: Claude Code, Codex, Gemini, Cursor, Copilot CLI, OpenCode,
  FactoryAI Droid, Pi, and Vogon.
- Canonicalized common aliases: `gemini-cli`, `cursor-cli`, and `copilot`.
- Kept native replay execution scoped to launchable agents while reporting
  session-only integrations as skipped eval rows with clear messages.
- The validator now checks status counts, pass rate, file averages, semantic
  averages, duration averages, risk score, and token totals against embedded
  replay runs.
- Added a negative check that rejects stale eval summaries.
- Kept explicit `--check` validation compatible with empty eval `runs[]`
  reports, while the built-in example still proves nested run behavior.
- Updated JSON schema, testing, acceptance, and command docs.

## Why It Matters

Replay Lab should be able to compare the whole Entire coder surface without
claiming integrations can run before they have a launch contract. This release
makes `--agent all` honest and useful for MVP evals while also proving checked
eval summary rows match the embedded replay runs they describe.
