# Entire Replay Lab v0.1.43

Legacy report array normalization.

## What Changed

- Replay and eval reports now normalize required JSON arrays before saving,
  reading, or printing JSON.
- Legacy `null` values for `files_touched`, `changed_files`, `agents`,
  `summaries`, and `runs` are re-emitted as empty arrays.
- Eval `summaries` is always serialized, including the empty-array case required
  by the schema.
- Added focused Go coverage for legacy replay and eval reports with null
  required arrays.
- Refreshed `patches/entire-replay-lab.patch` and the pinned patch SHA-256.
- Updated architecture, acceptance, testing, JSON schema, and reproducibility
  docs.

## Why It Matters

Report commands should be stable even when operators inspect older or manually
copied artifacts. This release keeps `entire replay report --json` and
`entire eval report --json` aligned with the schema instead of echoing legacy
null arrays back to downstream tooling.

## Known Limitations

- This remains a prototype patch repo, not an upstream Entire CLI release.
- Live replay still requires each selected coding agent to be installed and
  authenticated locally.
- Pi is included in all-agent eval output, but skipped until a safe
  non-interactive launch contract is available.
