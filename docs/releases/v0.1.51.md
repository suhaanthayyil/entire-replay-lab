# Entire Replay Lab v0.1.51

Legacy embedded run ID recovery.

## What Changed

- Eval report readers now recover missing embedded replay run IDs from the eval
  report ID and original run position.
- Existing embedded replay run IDs are preserved.
- Recovery composes with top-level eval ID recovery, so sparse legacy eval files
  can recover both the eval ID and run IDs from the filename.
- Strengthened Go coverage for eval reports with missing top-level IDs and
  missing embedded replay run IDs.
- Refreshed `patches/entire-replay-lab.patch` and the pinned patch SHA-256.
- Updated architecture, acceptance, testing, JSON schema, and reproducibility
  docs.

## Why It Matters

`ReplayEvalRun.runs[]` uses the same report shape as saved replay reports. When
a legacy eval report omitted an embedded run ID, rendered JSON could contain run
rows that were hard to cross-reference. Deriving a stable ID from the eval ID
and original run position keeps the report self-identifying without overwriting
any existing run IDs.

## Known Limitations

- This remains a prototype patch repo, not an upstream Entire CLI release.
- Live replay still requires each selected coding agent to be installed and
  authenticated locally.
- Pi is included in all-agent eval output, but skipped until a safe
  non-interactive launch contract is available.
