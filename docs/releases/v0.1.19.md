# Entire Replay Lab v0.1.19

Nested eval run schema validation.

## What Changed

- Updated `schemas/eval-run.schema.json` so `runs[]` validates against the
  ReplayRun schema instead of accepting any array item.
- Extended `scripts/validate-examples.py` with local cross-schema `$ref`
  resolution.
- Expanded the eval example with embedded replay runs, including recursive
  subagent token usage.
- Added a negative validator check that rejects undocumented fields inside
  embedded eval replay runs.
- Tightened report fixtures to prove `eval report --json` preserves embedded
  replay runs and recursive token usage.
- Updated JSON schema, testing, acceptance, and command docs.

## Why It Matters

Eval reports are the future dashboard payload for comparing agents. Before this
release, the schema validated the eval wrapper and summaries, but a malformed
embedded replay run could slip through. Now the same ReplayRun contract is
enforced whether a run is saved alone or nested inside an eval.
