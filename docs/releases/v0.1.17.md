# Entire Replay Lab v0.1.17

Closed JSON schema validation for checked Replay/Eval examples.

## What Changed

- Added `additionalProperties: false` to the ReplayRun and ReplayEvalRun schema
  objects that define stable v1 fields.
- Updated `scripts/validate-examples.py` to enforce
  `additionalProperties: false`.
- Updated JSON schema and testing docs to make schema/example updates part of
  the JSON contract.

## Why It Matters

Replay Lab reports are meant to feed future dashboards and comparisons. If
examples quietly accept undocumented fields, the public JSON contract can drift
without anyone noticing. This release makes schema drift visible during normal
repo, release, and CI validation.
