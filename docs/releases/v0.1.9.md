# Entire Replay Lab v0.1.9

Pinned patch hash verification for the Replay Lab artifact.

## What Changed

- Added `ENTIRE_REPLAY_DEFAULT_PATCH_SHA256` to `scripts/replay-lab-env.sh`.
- `verify-reproducibility.sh` now fails if the patch content does not match the
  pinned SHA-256.
- The verifier prints both the actual and expected patch hash for audit notes.
- Local experiments can intentionally disable the hash pin with
  `ENTIRE_REPLAY_PATCH_SHA256=`.
- Updated reproducibility docs and changelog.

## Why It Matters

Replay Lab is distributed as a patch against a pinned Entire CLI commit. The
base commit was already pinned, but the patch artifact itself could drift
without failing CI. This release makes the patch content part of the verified
release contract.
